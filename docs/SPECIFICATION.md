# メニュー管理システム仕様書

## 1. システム概要

### 1.1 目的
- ユーザーの体格と目標に合わせた最適な食事メニューを提供
- 栄養バランスの自動計算と調整
- 買い物リストの自動生成と管理

### 1.2 主要機能
- メニュータイプ別の食事プラン提供
- PFCバランスの自動計算
- 食材量の自動調整
- 買い物リスト管理
- お気に入りメニュー管理

## 2. メニュータイプ

### 2.1 食材固定版
- **目的**: 同じ食材を効率的に使い回すメニュー
- **特徴**:
  - 基本食材を固定
  - 調理方法のバリエーション
  - 買い物効率の最適化
- **データ構成**:
  - `fixedMenu.ts`: メニュー定義
  - `fixedIngredientsDatabase.ts`: 使用食材データベース

### 2.2 レシピ版
- **目的**: 本格的な料理レシピの提供
- **特徴**:
  - 詳細な調理手順
  - 多様な食材使用
  - 本格的な料理体験
- **データ構成**:
  - `recipeMenu/index.ts`: メニュー統合
  - `recipeMenu/{day}Menu.ts`: 曜日別メニュー
  - `recipeIngredientsDatabase.ts`: レシピ用食材データベース

### 2.3 コンビニ版
- **目的**: 市販品を活用した時短メニュー
- **特徴**:
  - コンビニ商品の活用
  - 調理時間の最小化
  - 手軽な食材調達
- **データ構成**:
  - `convenienceMenu.ts`: メニュー定義
  - `foodDatabase.ts`: コンビニ商品データベース

## 3. データ構造

### 3.1 メニューデータ構造
```typescript
interface MenuItem {
  name: string;                    // メニュー名
  pfc: {                          // 栄養成分
    protein: number;              // タンパク質(g)
    fat: number;                  // 脂質(g)
    carbs: number;               // 炭水化物(g)
  };
  difficulty?: 'easy' | 'medium' | 'hard';  // 難易度
  cookingTime?: number;           // 調理時間（分）
  servings?: number;              // 何人分
  ingredients?: Ingredient[];     // 材料リスト
  instructions?: string[];        // 調理手順
  detailedInstructions?: string[]; // 詳細手順
  timing?: '朝食' | '昼食' | '夕食';  // 食事タイミング
  adjustedPortions?: boolean;     // 分量調整済みフラグ
}
```

### 3.2 食材データ構造
```typescript
interface FoodItem {
  name: string;      // 食材名
  size: string;      // 基準量
  pfc: {
    protein: number; // タンパク質(g)
    fat: number;     // 脂質(g)
    carbs: number;   // 炭水化物(g)
    calories: number;// カロリー(kcal)
  };
  unit: string;      // 単位
  notes?: string;    // 代替品情報
}
```

## 4. 主要機能詳細

### 4.1 栄養計算機能
- ユーザープロフィールに基づく必要栄養素の計算
- 1日の目標カロリーとPFCバランスの算出
- メニューごとの栄養価の自動計算

### 4.2 分量調整機能
- ユーザーの必要栄養素に基づく食材量の自動調整
- パック商品や個数単位の商品は調整対象外
- 調整比率の上限設定（最大2倍まで）

### 4.3 買い物リスト機能
- メニューからの自動リスト生成
- カテゴリー別の食材分類
- チェックリスト機能
- 複数日分のまとめ買いリスト作成

### 4.4 お気に入り機能
- メニューのお気に入り登録
- お気に入りメニューの管理
- コンテキスト情報の保存（パターン、曜日、タイミング）
